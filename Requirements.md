# Requirements outline for Kudae.

The project has been primarily divided on three lines of focus. 
- Relief Camps 
- Rescue Operations 
- Communication and Logistics

Each of these would have different web-views so as to give an effective overview of the battlefront.

Each focus line would have user-management at various levels to help delegate tasks. Incorporation of tools and technologies to help coordinate and manage these users would be effective.

Overview of version 0.0.1 of the project
1) Relief Camps
    1) Camp details: Address,How to reach, Communication modes and numbers
    2) Camp volunter details: Name,contact number, designation
    3) Camp accomodation and stock status (Visible to admins of different camps only. This is a sensitive data and if it is not secure to reveal this, the admin should be able to disable sharing this statistic. )
    4) Camp occupant details (sensitive data, privacy controls are require to restrict view from govt. agencies or other user groups )
    5) Camp infrastructure details: electricity, Communication modes, vehicles available. Statistics and status of each
    6) Camp requirement and announcements

2) Rescue Operations
    1) Rescue operator's number
    2) Rescue supply/demand status
    3) Rescue requirements and announcements
    4) Rescue requests from victims with status
    5) Rescue team comunication with command and control
    6) Command and control dashboard

3) Communication and Logistics
    1) Telecom failsafe : Information on nearest telephone or internet facilties
    2) Roadblocks or dangerous routes: Information on roadblocks
    3) Requirement management: Repair, provision and support board to post details of downed communications with priority.
    4) Emergency transport management: Information on available transports.
